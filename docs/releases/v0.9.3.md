# Topo v0.9.3

Topo v0.9.3 is the consolidated Debian and RPM package distribution release. It supersedes the exploratory v0.9.1 and v0.9.2 package releases with a cleaner final shape for GitHub Release installers, checksums, GPG verification, and package-manager lifecycle behavior.

The existing one-line script installer remains supported. The new `.deb` and `.rpm` packages provide a system package path for users who prefer installation through `apt` or `dnf` from downloaded GitHub Release assets.

## Highlights

### Debian and RPM Packages

- Added GitHub Release installers for Debian/Ubuntu and Fedora/RHEL-family systems.
- Packages install Topo into `/usr/lib/topo`.
- Packages expose the command through `/usr/bin/topo`.
- Packages include the matching architecture-specific Rust engine binary.
- Package staging removes Python cache artifacts such as `__pycache__`, `.pyc`, `.pyo`, and `$py.class`.

Release package names:

```text
topo_0.9.3_amd64.deb
topo_0.9.3_arm64.deb
topo-0.9.3-1.x86_64.rpm
topo-0.9.3-1.aarch64.rpm
```

Architecture naming follows each package ecosystem:

- Debian/Ubuntu: `amd64`, `arm64`
- Fedora/RHEL RPM: `x86_64`, `aarch64`
- RPM `-1` is the package release/iteration, not the Topo upstream version.

### Installation Paths

Script installation remains available:

```bash
curl -fsSL https://raw.githubusercontent.com/Jesencloud/Topo/main/install.sh | bash
```

GitHub Release package installation is now available:

```bash
sudo apt install ./topo_0.9.3_amd64.deb
sudo dnf install ./topo-0.9.3-1.x86_64.rpm
```

For ARM64 systems, use:

```bash
sudo apt install ./topo_0.9.3_arm64.deb
sudo dnf install ./topo-0.9.3-1.aarch64.rpm
```

### Install Source Separation

- Script installs keep the `~/.topo` layout and continue to use Topo's GitHub-based update/remove flow.
- Package installs use `/usr/lib/topo` and are marked with `/usr/lib/topo/.topo-install-source`.
- Package-mode `topo update` and `topo remove` do not run the GitHub installer and do not delete package-owned files directly.
- Package-mode lifecycle operations are delegated to the system package manager.

Distro-aware package-mode guidance:

```bash
# Debian/Ubuntu
sudo apt upgrade topo
sudo apt remove topo

# Fedora/RHEL
sudo dnf upgrade topo
sudo dnf remove topo
```

Unknown systems fall back to showing both common package-manager command families.

### Compact Release Assets

GitHub Release assets are intentionally kept compact. Instead of uploading one `.sha256`, one `.asc`, and one `.sha256.asc` sidecar per package, the release now uploads one checksum manifest and one detached signature for that manifest.

Expected user-facing Release assets:

```text
topo_0.9.3_amd64.deb
topo_0.9.3_arm64.deb
topo-0.9.3-1.x86_64.rpm
topo-0.9.3-1.aarch64.rpm
topo-core-x86_64
topo-core-aarch64
SHA256SUMS
SHA256SUMS.asc
topo-release-public.asc
```

`SHA256SUMS` records final Release asset filenames only, so users can download assets into one directory and run:

```bash
sha256sum -c SHA256SUMS
```

### GPG-Signed Checksum Manifest

- The release workflow imports the maintainer GPG key from GitHub Actions secrets.
- The release workflow signs `SHA256SUMS` with an armored detached signature.
- The release uploads `SHA256SUMS.asc` and `topo-release-public.asc`.
- The public key fingerprint is:

```text
4B35 C17C F8E6 6373 2726  A99F 5008 6DB9 98B4 D883
```

Manual verification flow:

```bash
curl -fsSLO https://github.com/Jesencloud/Topo/releases/latest/download/topo-release-public.asc
gpg --import topo-release-public.asc
gpg --fingerprint "Topo Release"
gpg --verify SHA256SUMS.asc SHA256SUMS
sha256sum -c SHA256SUMS
```

`SHA256SUMS.asc` verifies the checksum manifest. It does not replace RPM package signatures or future APT/DNF repository metadata signing.

### Release Automation

- GitHub Actions builds `topo-core` for `x86_64` and `aarch64`.
- The package job builds all Debian and RPM installers with `fpm`.
- Ubuntu smoke tests install the generated `amd64` `.deb` with `sudo apt install`.
- Fedora smoke tests install the generated `x86_64` `.rpm` with `dnf install`.
- Smoke tests verify:
  - `topo --version`
  - `/usr/lib/topo/.topo-install-source`
  - package-mode `topo update`
  - package-mode `topo remove`
- Release assets are attached only after package build and smoke-test jobs pass.
- The release job stages final uploads into `release-assets/`, generates `SHA256SUMS`, signs the manifest, and uploads only the compact asset set.

## 中文重点

Topo v0.9.3 是正式整理后的 Debian/RPM 包分发版本。它把 v0.9.1 和 v0.9.2 中逐步摸索出来的包构建、安装来源区分、系统包管理器生命周期提示、Release 校验清单和 GPG 签名合并成一个完整方案。

### 重点更新

- **新增系统安装包**：GitHub Release 提供 Debian/Ubuntu 的 `.deb` 和 Fedora/RHEL 的 `.rpm`。
- **保留脚本安装**：`curl | bash` 继续支持，脚本安装仍使用 `~/.topo` 和原有 GitHub 更新/卸载逻辑。
- **区分安装来源**：包安装写入 `/usr/lib/topo/.topo-install-source=package`，脚本安装写入 `script`。
- **生命周期交给包管理器**：包安装后，`topo update` / `topo remove` 会提示 `apt` 或 `dnf`，不会直接运行 GitHub installer 或删除系统包文件。
- **Release 附件精简**：不再上传每个文件的 `.sha256`、`.asc`、`.sha256.asc`，只上传 `SHA256SUMS` 和 `SHA256SUMS.asc`。
- **GPG 校验清单签名**：`SHA256SUMS.asc` 用于验证 `SHA256SUMS`，公钥随 Release 以 `topo-release-public.asc` 上传。
- **CI 冒烟测试**：Ubuntu 安装 `.deb`，Fedora 安装 `.rpm`，并验证版本、安装来源、更新提示和卸载提示。
