# Topo v0.9.2

Topo v0.9.2 completes the next step in the GitHub Release package distribution path: Release assets now support detached GPG signatures, the public release key is published with each release, and the README documents both script and package installation paths.

## Highlights

### Signed Release Assets

- Added GPG detached signature generation to the GitHub Actions release job.
- The release workflow now signs:
  - raw `topo-core` engine binaries
  - `.deb` packages
  - `.rpm` packages
  - `.sha256` checksum files
- Signatures are uploaded as armored `.asc` files alongside the original assets.
- Release signing uses GitHub Actions secrets:
  - `GPG_PRIVATE_KEY`
  - `GPG_PASSPHRASE`

### Published Release Public Key

- Added `assets/topo-release-public.asc`.
- The public key is uploaded as a GitHub Release asset so users can verify detached `.asc` signatures.
- Topo release key fingerprint:

```text
4B35 C17C F8E6 6373 2726  A99F 5008 6DB9 98B4 D883
```

### Package Installation Documentation

- Updated the README Quick Start section to document both installation paths:
  - script install via `curl | bash`
  - GitHub Release package install via `.deb` / `.rpm`
- Package install examples now use v0.9.2 asset names:

```bash
sudo apt install ./topo_0.9.2_amd64.deb
sudo dnf install ./topo-0.9.2-1.x86_64.rpm
```

- ARM64 package names are documented for Debian/Ubuntu and Fedora/RHEL.
- README intentionally keeps advanced checksum/GPG verification commands out of the main install flow; those details remain in the development report.

### Release Workflow Polish

- The release job now uploads `.asc` signatures for both package files and raw engine assets.
- The release job uploads `topo-release-public.asc`.
- Package build and smoke-test jobs remain release gates, so package assets are published only after installation checks pass.

### Asset Cleanup

- Removed the obsolete `assets/topo_home.png` file.
- The current README no longer references `assets/topo_home.png`; `assets/topo.png` remains the primary visual asset.

## Verification

- Parsed `.github/workflows/build-engine.yml` with Ruby YAML loading.
- Ran `git diff --check`.
- Verified `assets/topo-release-public.asc` imports successfully in a clean temporary GPG home.
- Confirmed no PGP private key material is present in the repository.

## Notes

- Detached `.asc` signatures are for manual GitHub Release asset verification.
- These signatures do not replace RPM package signatures or APT/DNF repository metadata signing.
- First-party APT/DNF repositories should wait until several GitHub Release package versions have proven stable.
- Official Debian/Fedora repository submission should come after the package format and lifecycle behavior are mature.
