# Topo v0.6.0

Deep clean, inspect, and maintain your Linux system.

This is the first public release of Topo. The focus of this release is safety, Linux-native application management, and a release/update path that can be trusted beyond a local development machine.

Topo is still a young project, but v0.6.0 is now ready for public testing as a practical Linux cleanup, disk analysis, and app uninstall tool.

## Highlights

### Safer deletion layer

- Added a deletion audit log at `~/.local/state/topo/deletions.log`.
- Recorded destructive operation outcomes including dry-runs, skipped paths, Trash moves, permanent deletes, Trash fallback failures, and failed deletes.
- Added a centralized deletion validator used by `safe_remove()`, analyze deletion, uninstall residue cleanup, and clean tasks.
- Blocked dangerous Linux targets such as `/`, `/bin`, `/boot`, `/dev`, `/proc`, `/sys`, `/run`, `/home`, `/var/lib`, `/etc/passwd`, and traversal variants.
- Added a Linux dangerous-path fuzz corpus and regression tests.
- Added sensitive data protection for SSH/GPG, keyrings, password managers, browser profiles, input methods, wallets, database clients, and IDE/editor configuration.

### Cleanup history

- Added `topo history`.
- Grouped cleanup and uninstall operations by session.
- Summarized removed, trashed, skipped, failed, and reclaimed size totals.
- Recorded uninstall package events even when no residue paths are found.

### Linux-native uninstall support

- Expanded package-manager coverage across DNF/RPM, APT/dpkg, Pacman, Flatpak, and Snap.
- Added package-manager-specific removal commands so non-Fedora systems no longer fall through to DNF.
- Filtered kernel, driver, desktop shell, display stack, audio/network stack, and other system components from the normal uninstall list.
- Refined GNOME, IBus, and LibreOffice package visibility:
  - Hide GNOME core components, system utilities, and input-method engine packages.
  - Hide internal LibreOffice packages such as `libreoffice-core` and `libreoffice-xsltfilter`.
  - Keep user-facing apps such as `gnome-calculator`, `gnome-calendar`, `libreoffice-writer`, `libreoffice-calc`, and `libreoffice-impress`.
- Added official-only residue policy for high-risk software such as VPN/security tools, input methods, password managers, and SSH/GPG-related tools.

### Disk analysis and cache detection

- Added valid `CACHEDIR.TAG` detection.
- Marked cache-tagged directories as cleanable in analyze results.
- Reused the same `CACHEDIR.TAG` detection from clean tasks.
- Continued using the Rust `topo-core` engine for fast disk scanning.

### Install, update, and release delivery

- Changed `topo update` to read GitHub Releases/latest instead of tracking the development branch.
- Added `install.sh --version` and `install.sh --ref` support for installing a specific tag.
- Added installer checks for Python's `packaging` module, with install hints for Fedora/RHEL, Debian/Ubuntu, and Arch/Manjaro.
- Improved launcher linking:
  - root installs link to `/usr/local/bin/topo`
  - regular user installs link to `~/.local/bin/topo`
- Added GitHub Actions release assets for:
  - `topo-core-x86_64`
  - `topo-core-x86_64.sha256`
  - `topo-core-aarch64`
  - `topo-core-aarch64.sha256`

## Install

```bash
curl -fsSL https://raw.githubusercontent.com/Jesencloud/Topo/main/install.sh | bash
```

After installation:

```bash
topo --help
topo status
topo clean
topo analyze
topo uninstall
topo history
```

## Verified environments

- Fedora local development machine
- Ubuntu 24.04 Podman container
- Arch Linux Podman container

Target support:

- Fedora/RHEL/CentOS family
- Ubuntu/Debian/Mint/Pop family
- Arch/Manjaro/EndeavourOS family
- x86_64
- aarch64 / arm64

## Notes

- This is the first public release. Start with non-destructive commands such as `topo status`, `topo analyze`, and `topo history`.
- `topo clean` and `topo uninstall` include multiple safety checks and deletion audit logging, but you should still review interactive confirmation screens carefully.
- Cleanup and uninstall history is stored at `~/.local/state/topo/deletions.log`.
- If package-manager detection or uninstall filtering looks wrong, please report your distribution, package name, and `topo --version`.

## Next

- Align the default install path more tightly with latest stable releases.
- Add automatic `.sha256` verification in `install.sh`.
- Add `topo doctor` for dependency and environment checks.
- Expand real desktop-environment validation for Fedora/RHEL, Ubuntu/Debian, and Arch/Manjaro.
- Add an advanced uninstall mode for showing hidden system components.

### 重点更新

- 新增删除审计日志：记录 dry-run、跳过、Trash、永久删除、失败等结果。
- 新增统一删除路径校验：阻止 `/`、`/bin`、`/boot`、`/dev`、`/proc`、`/sys`、`/run`、`/home`、`/var/lib`、`/etc/passwd` 等危险路径。

- 扩展 DNF/RPM、APT/dpkg、Pacman、Flatpak、Snap 的扫描和卸载路径。
- 优化卸载列表过滤：隐藏 kernel、driver、GNOME 核心组件、IBus 输入法引擎、LibreOffice 内部包等高风险或内部组件。
- 保留用户可理解的应用项，例如 `gnome-calculator`、`libreoffice-writer`、`libreoffice-calc`、`libreoffice-impress`。
- 支持合法 `CACHEDIR.TAG` 识别，提升 analyze 和 clean 的缓存判断准确性。
- `topo update` 改为基于 GitHub Releases/latest，不再跟随 `main` 开发分支。
- GitHub Actions 会为 release 构建并上传 x86_64 和 aarch64 的 `topo-core` 二进制。
