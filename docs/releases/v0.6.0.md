# Topo v0.6.0

Topo 的首个公开 Release。

这版把重点放在三件事上：清理删除的安全边界、Linux 原生的应用卸载体验，以及可稳定分发的 release/update 链路。它已经从单机脚本推进到一个可以公开试用的 Linux 清理、分析和卸载工具。

## 主要更新

### 更安全的删除层

- 增加删除审计日志，记录删除、跳过、失败、dry-run、Trash 失败回退等事件。
- 新增统一路径校验，阻止删除 `/`、`/bin`、`/boot`、`/dev`、`/proc`、`/sys`、`/run`、`/home`、`/var/lib`、`/etc/passwd` 等危险路径。
- 加入 Linux 危险路径 fuzz 语料和回归测试。
- 统一 `safe_remove()`、analyze 删除、uninstall 残留清理和 clean 清理的保护逻辑。
- 增加敏感应用数据保护，包括 SSH/GPG、keyring、密码管理器、浏览器 profile、输入法、钱包、数据库客户端和 IDE 配置。

### 清理历史

- 新增 `topo history`。
- 支持按 session 汇总 removed、trashed、skipped、failed 和释放空间。
- uninstall 即使没有残留目录，也会记录包卸载历史。

### Linux 原生卸载体验

- DNF/RPM、APT/dpkg、Pacman、Flatpak、Snap 扫描和卸载路径进一步对齐。
- 卸载列表过滤系统组件，避免展示 kernel、driver、desktop shell、display stack、audio/network stack 等高风险包。
- 增强 GNOME、IBus、LibreOffice 等包的展示规则：
  - 隐藏 GNOME 核心组件、系统工具和输入法引擎包。
  - 隐藏 `libreoffice-core`、`libreoffice-xsltfilter` 等内部包。
  - 保留 `gnome-calculator`、`gnome-calendar`、`libreoffice-writer`、`libreoffice-calc`、`libreoffice-impress` 等用户可理解的应用项。
- 高风险软件采用官方卸载优先策略，VPN、安全工具、输入法、密码管理器、SSH/GPG 相关工具默认不清理残留数据。

### Disk Analyze 与缓存识别

- analyze 支持识别合法 `CACHEDIR.TAG`，并把对应目录标记为可清理缓存。
- clean 侧复用同一套 `CACHEDIR.TAG` 判断逻辑。
- Rust `topo-core` 继续作为高速扫描引擎使用。

### 安装、更新与 Release

- `topo update` 改为基于 GitHub Releases/latest tag，不再跟随 `main` 分支。
- `install.sh` 支持 `--version` / `--ref`，可以安装指定 tag。
- 安装脚本新增 `python packaging` 依赖检查，并给出 Fedora、Ubuntu/Debian、Arch/Manjaro 安装提示。
- root 安装时命令入口默认链接到 `/usr/local/bin/topo`，普通用户仍使用 `~/.local/bin/topo`。
- GitHub Actions 会在 tag push 后构建并上传：
  - `topo-core-x86_64`
  - `topo-core-x86_64.sha256`
  - `topo-core-aarch64`
  - `topo-core-aarch64.sha256`

## 安装

```bash
curl -fsSL https://raw.githubusercontent.com/Jesencloud/Topo/main/install.sh | bash
```

安装完成后：

```bash
topo --help
topo status
topo clean
topo analyze
topo uninstall
topo history
```

## 已验证环境

- Fedora 本机开发环境
- Ubuntu 24.04 Podman 容器
- Arch Linux Podman 容器

支持目标：

- Fedora/RHEL/CentOS family
- Ubuntu/Debian/Mint/Pop family
- Arch/Manjaro/EndeavourOS family
- x86_64
- aarch64 / arm64

## 注意事项

- 这是首个公开 release，建议先从 `topo status`、`topo analyze`、`topo history` 等非破坏性命令开始体验。
- `topo clean` 和 `topo uninstall` 已加入多层保护和审计日志，但仍建议仔细阅读交互式确认页。
- 清理和卸载历史位于 `~/.local/state/topo/deletions.log`。
- 如果发现包管理器适配或卸载列表过滤不准确，建议带上发行版、包名和 `topo --version` 反馈。
