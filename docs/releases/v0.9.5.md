# Topo v0.9.5

Topo v0.9.5 focuses on safer cache cleanup and clearer Analyze Disk behavior. It consolidates browser, desktop-app, `CACHEDIR.TAG`, and generic XDG cache rules so Clean and Analyze share the same classification model while keeping destructive cleanup conservative.

## Highlights

### Safer Browser Cache Cleanup

- Added a shared browser cache definition layer for Firefox-family and Chromium-family browsers.
- Browser cache cleanup now lives in Clean through `clean_browser_caches()`.
- Analyze keeps a guarded fallback for manually selected browser profile roots.
- Browser profile roots remain protected; Topo does not delete whole profiles such as `.config/google-chrome`, `.mozilla`, or profile directories like `Default`.
- Browser credential and state files remain protected, including `Login Data`, `Cookies`, `logins.json`, `key4.db`, and `cookies.sqlite`.
- Known cache children such as `Cache`, `Code Cache`, `GPUCache`, `ShaderCache`, `cache2`, `startupCache`, and `Service Worker/CacheStorage` can be cleaned safely.

### Shared Cache Classification

- Added shared cache discovery in `src/core/app_cache.py` for Clean and Analyze.
- Added desktop app cache definitions in `src/core/desktop_app_cache.py`.
- Removed the old duplicated desktop/browser cache entries from the generic app cleanup path.
- Analyze now uses shared cache metadata to mark cleanable entries with reasons such as:
  - `App cache`
  - `CACHEDIR.TAG`
  - `XDG cache`
- Clean reuses the same discovery logic for browser caches, desktop app caches, tagged caches, and generic XDG cache candidates.

### Conservative Deletion Boundaries

- Symlinked browser cache directories are skipped before cache-name matching, so cleanup cannot follow a profile-local symlink into external data.
- Symlinked cleanup roots passed directly to `clean_app_generic()` are skipped before path resolution.
- `CACHEDIR.TAG` detection rejects symlinked directories and symlinked tag files.
- Generic XDG cache discovery skips symlinked `~/.cache/<name>` entries.
- WeChat user data is no longer cleaned:
  - `.xwechat`
  - `.var/app/com.tencent.WeChat/config/xwechat`
  - `Documents/WeChat Files`
- Only the explicit WeChat Flatpak cache path remains in Clean definitions.

### Faster Clean Size Accounting

- Clean now uses the Rust engine for fast cache size accounting.
- `clean_app_generic()` scans a cache parent once and reuses direct child sizes when calling `safe_remove()`.
- If the Rust engine is unavailable or returns incompatible data, Topo falls back to per-child sizing.
- Fast child-size parsing is centralized in `src/core/file_ops.py` with `get_direct_child_sizes_fast()`.

### Analyze Disk Polish

- Cleanable cache entries no longer use the `🧹` icon in Analyze Disk, avoiding the impression that every marked cache entry will be automatically cleaned.
- Analyze Disk keeps regular directory/file icons while preserving `is_cleanable` and `cleanable_reason` metadata.
- Added extra visual spacing between the `🗂️` folder icon and the folder name in Analyze Disk lists and selected-item summaries.
- Analyze progress text is now consistent with the Rust engine scan message.

### App Removal Sorting

- The `Select Application to Remove` view now sorts by install time by default.
- Recently installed apps appear first.
- Missing install times are handled safely, with size used as a secondary key.

## Notes

- Clean remains the primary place for automated cache cleanup.
- Analyze marks cache-like entries and provides guarded deletion behavior only where validation allows it.
- Whole browser profile deletion remains blocked.
- Future cache categories such as package-manager caches, container caches, and AI/model caches should continue to share Analyze metadata where useful while keeping command-based cleanup in Clean.

## 中文重点

Topo v0.9.5 是缓存清理安全和 Analyze Disk 体验优化版本。重点是让 Clean 和 Analyze 共用缓存识别规则，同时继续保持保守删除边界。

### 重点更新

- **浏览器缓存清理更安全**：支持 Firefox/LibreWolf/Chrome/Chromium/Brave/Edge/Vivaldi/Opera 等浏览器缓存识别；只清理明确的缓存子目录，不删除整个浏览器 profile。
- **浏览器敏感数据继续保护**：`Login Data`、`Cookies`、`logins.json`、`key4.db`、`cookies.sqlite` 等不会被当作缓存删除。
- **Clean 和 Analyze 共用缓存规则**：浏览器缓存、桌面应用缓存、`CACHEDIR.TAG`、通用 XDG cache 都通过共享逻辑识别，减少硬编码和重复代码。
- **软链接安全修复**：浏览器缓存目录、通用 XDG cache、`CACHEDIR.TAG` 和直接传入 Clean 的 cache root 都增加了 symlink 防护，避免跟随软链接删除外部数据。
- **WeChat 用户数据保护**：`.xwechat`、Flatpak `config/xwechat`、`Documents/WeChat Files` 不再被 Clean 删除，只保留明确的 cache 路径。
- **Clean 统计更快**：缓存父目录只启动一次 Rust 扫描并复用子项大小，避免大量小文件时频繁启动 Rust 子进程。
- **Analyze Disk 显示优化**：去掉容易误解的 `🧹` 图标，保留普通文件夹/文件图标，并在 `🗂️` 和文件夹名称之间增加视觉空格。
- **应用卸载列表排序优化**：默认按安装时间排序，最近安装的应用优先显示。

### 后续方向

- Clean 继续负责自动清理。
- Analyze 继续负责展示、识别和受保护的手动删除。
- 后续如果加入包管理器缓存、容器缓存、AI/model 缓存，应优先复用共享元数据，但具体清理动作仍应放在 Clean。
