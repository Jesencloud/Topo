# Topo v0.9.1

Topo v0.9.1 is the first package-manager distribution release. It adds GitHub Release `.deb` and `.rpm` installers while keeping the existing one-line `curl | bash` installer fully supported.

## Highlights

### Debian and RPM Packages

- Added automated `.deb` and `.rpm` package generation for GitHub Releases.
- Packages install Topo into `/usr/lib/topo` and expose `/usr/bin/topo`.
- Package builds include the matching architecture-specific Rust engine:
  - `topo_${VERSION}_amd64.deb`
  - `topo_${VERSION}_arm64.deb`
  - `topo-${VERSION}-1.x86_64.rpm`
  - `topo-${VERSION}-1.aarch64.rpm`
- Python cache artifacts such as `__pycache__`, `.pyc`, `.pyo`, and `$py.class` are removed from package staging.

### Script and Package Install Separation

- Existing script installs continue to use:
  ```bash
  curl -fsSL https://raw.githubusercontent.com/Jesencloud/Topo/main/install.sh | bash
  ```
- Script installs keep the existing `~/.topo` layout and GitHub-based `topo update` / `topo remove` flow.
- Package installs are marked with `/usr/lib/topo/.topo-install-source`.
- Package installs delegate lifecycle operations to the system package manager instead of running the GitHub installer or deleting package-owned files directly.

### Distro-Aware Lifecycle Guidance

- On Ubuntu/Debian-family systems, package-mode commands now show only:
  ```bash
  sudo apt upgrade topo
  sudo apt remove topo
  ```
- On Fedora/RHEL-family systems, package-mode commands now show only:
  ```bash
  sudo dnf upgrade topo
  sudo dnf remove topo
  ```
- Unknown systems fall back to showing both common command families.

### Checksums and Release Automation

- Every generated `.deb` and `.rpm` now receives a sibling `.sha256` file.
- GitHub Release assets now include package files and checksum files.
- The GitHub Actions release workflow now has a dedicated package build job.
- Release publishing is gated behind package smoke tests.

### Package Smoke Tests

- Added Ubuntu smoke testing for the generated `amd64` `.deb`:
  - `sudo apt install ./topo_..._amd64.deb`
  - `topo --version`
  - `cat /usr/lib/topo/.topo-install-source`
  - `topo update`
  - `topo remove`
- Added Fedora smoke testing for the generated `x86_64` `.rpm` in a Fedora 44 container:
  - `dnf install ./topo-...x86_64.rpm`
  - `topo --version`
  - `cat /usr/lib/topo/.topo-install-source`
  - `topo update`
  - `topo remove`
- GitHub Releases attach package assets only after both smoke-test jobs pass.

## Verification

- `bash -n install.sh`

- `ruff format --check .`
- `env HOME=/tmp/topo_pytest_home pytest -q` - 208 passed
- `sha256sum -c *.sha256` inside `dist/packages` - all package checksums passed

## Notes

- Debian package architecture names use `amd64` and `arm64`.
- RPM package architecture names use `x86_64` and `aarch64`.
- The RPM `-1` suffix is the package release/iteration, not the Topo upstream version.
- This release is still GitHub Release package distribution. First-party APT/DNF repositories and signed repository metadata remain future work.

## 中文重点

Topo v0.9.1 是第一个面向包管理器分发的版本。它在保留原有 `curl | bash` 安装方式的同时，为 GitHub Release 增加 `.deb` 和 `.rpm` 安装包。

### 重点更新

- **新增 Debian/RPM 安装包**：Release 会产出 `amd64`、`arm64`、`x86_64`、`aarch64` 四类包。
- **保留脚本安装**：`curl -fsSL ... | bash` 继续安装到 `~/.topo`，并继续使用现有 GitHub 更新/卸载逻辑。
- **区分安装来源**：包安装写入 `.topo-install-source=package`，脚本安装写入 `script`。
- **生命周期交给系统包管理器**：包安装后，`topo update` / `topo remove` 不再执行 GitHub installer 或直接删除系统包文件。
- **按发行版显示正确命令**：Ubuntu/Debian 只提示 `apt`，Fedora/RHEL 只提示 `dnf`。
- **新增校验文件**：每个 `.deb` / `.rpm` 都会生成同名 `.sha256`。
- **新增 CI 冒烟测试**：GitHub Actions 会在 Ubuntu 和 Fedora 环境中安装包并验证 `topo --version`、安装来源标记、更新提示和卸载提示。

### 后续方向

- 为包和校验文件增加 GPG 签名。
- 建立一方 APT/DNF 仓库，让用户最终可以使用 `sudo apt install topo` 或 `sudo dnf install topo`。
- 长期评估进入 Debian/Fedora 官方仓库。
